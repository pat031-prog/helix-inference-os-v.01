# Emergent Behavior Observatory v1

## Claim Boundary

This is a qualitative cloud-only observation run. It does not claim model
sentience, persistent internal model memory, local `.hlx` bit identity, or
numerical KV<->SSM state transfer.

## Inspiration

The format is inspired by qualitative system-card sections such as "Other
noteworthy behaviors and anecdotes": observations that are interesting but not
benchmarks.

## Null Hypothesis

In a multi-model signed-memory loop, qualitative observations cannot be tied to
specific evidence; anecdotes will be unsupported, over-anthropomorphic, or
causally ambiguous.

## Alternative Hypothesis

A signed-memory loop can produce qualitative observations that are explicitly
tied to turn IDs, memory IDs, parent-hash lineage, and claim-boundary caveats.

## Setup

- Four cloud models rotate through a signed-memory notebook.
- Each turn is inserted as a signed memory in one parent-hash chain.
- A signed anthropomorphic lure is present as semantically invalid evidence.
- An unsigned anecdote lure is inserted but must be absent from strict
  retrieval.
- An analyst model extracts noteworthy behaviors with citations.
- An auditor model checks citation discipline and claim boundaries.

## Pass Criteria

- Four distinct models are configured and all participate.
- At least eight total rounds run.
- All turn memories have `signature_verified=true`.
- The main parent-hash chain validates.
- The unsigned lure is absent from strict retrieval.
- The analyst returns parseable JSON.
- At least three noteworthy behaviors are reported.
- Every behavior cites real turn IDs and memory IDs.
- Method caveats are present.
- The analyst observes the claim boundary and avoids unqualified sentience
  claims.
- The auditor returns JSON verdict `pass` with no gate failures.

## Controls

- The signed lure tests semantic rejection despite valid provenance.
- The unsigned lure tests strict retrieval admission.
- The analyst is forced to cite concrete evidence rather than invent anecdotes.

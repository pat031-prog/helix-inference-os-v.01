Como correr el runtime real
  1. Clona el repo y crea un entorno Python con las dependencias del backend.
  2. Levanta el backend local con el entrypoint actual de Helix para exponer /assistants y /assistants/chat/stream.
  3. Abre la app local o conecta un frontend privado a esa URL de backend.
Por que esta instancia es distinta

Vercel sirve esta superficie como microsite publica estatica. Eso nos deja publicar la narrativa, los SVG, la evidencia y el browser de JSONs sin arrastrar el runtime pesado de modelos a la web publica.

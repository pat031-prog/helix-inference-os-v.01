HeliX is an Inference OS

HeliX is an Inference OS: it preserves computed model state below the prompt, shares semantic memory above the prompt, and schedules models as ephemeral workers.

This page was assembled by a deterministic control plane. The models suggest content, but HeliX owns continuity, audit, memory and routing.

01 / Ephemeral execution

Active Model

The model loaded right now is just the compute tip of the iceberg. HeliX can swap it out without confusing it with long-term memory.

02 / Below the prompt

Private .hlx State

KV cache and recurrent state are serialized as model-specific private work. A Qwen session never becomes a Zamba session.

03 / Above the prompt

Shared hmem

Observations, tool outputs, decisions and handoffs become portable context that any compatible agent can read.

04 / The conductor

Multimodel Scheduler

HeliX decides what wakes up, what sleeps, which state is restored and which memory context enters the prompt.
